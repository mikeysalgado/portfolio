Emotion Recognition Model

The emotion recognition model was a team-based project created for the Data Science Practicum course. We developed a machine learning model utilizing the EfficientNet-B0 architecture and compared its accuracy against two other models on the AffectNet facial emotion dataset. This was then implemented into a real-time video stream where users could walk up to the TV display and have their facial emotions predicted.

For this project, I applied my experience in machine learning and created the proof-of-concept process that connected back-end classification model to a front-end camera system using the OpenCV library for computer vision. One constraint that our team faced was having to develop this program from the ground up including technical specifications to full deployment within a 10-week span. We were able to find success by applying the Agile development which assisted in planning and developing features. This project has been carried over and expanded into an ongoing senior design project where we are developing an educational dashboard with a photobooth module featuring the emotion recognition model.

Spotify Music Genre Classifier

The Spotify genre predictor project was a 2-week individual project that was created for a Data Science course. This project consisted of exploratory data analysis and applying multiple prediction algorithms to achieve the highest accuracy on classifying songs based on 10 genres. At the end, a random-forest classifier was able to achieve a 56% accuracy, with a high of 84% accuracy for the classical genre.

As this was an individual project, I was responsible for researching a dataset and performing techniques that we had learned and practiced throughout the class such as data clean up, preprocessing, finding correlations, and training models for predictions. I explored classification approaches of kNN, decision tree, random forest, and stochastic gradient descent. One challenge that I faced during this project was fully understanding the dataset and preparing it for use in training as there were a mix of categorical variables, continuous variables, and entries that had null or placeholder values. I was able to apply my experience in machine learning and transform the data into useable formats such as one-hot encoding for variables such as key. In the future, I would like to revisit this and apply more models such as neural networks as learned in a Deep Learning course to improve accuracies.

UX Game Project

The MSOE Cooking app was a team-based project created for the Foundations of User Experience course. Our application was designed with a focus on MSOE students and their dietary needs to help them learn recipes that they can make with just the ingredients and appliances they have at home. The goal of this project was to apply concepts of design thinking such as gamification principles, user flow and personas, and developing wireframes to showcase our UI concepts.

My role on this team was leading group meetings and contributing to the group’s assignments by creating the team charter, Mural and Figma projects, user flow and journey, and developing the ingredients and recipe screens for the prototype. The final deliverable was a design brief containing each aspect of the project including interview notes and mockups throughout a 3-week timeline. One challenge that we had was learning to use UX software such as Figma, which I had used previously but gained more experience throughout the project.
